# Skill Template

This file defines the exact structure and content format for the SKILL.md and reference files generated by ui-cloner. Use these templates verbatim, filling in the extracted data.

---

## SKILL.md Template

```markdown
---
name: {product-name}-ui
description: "UI design system skill for {product-name}. Captures the complete visual language extracted from {N} screenshots: {N} design tokens, {N} components, {style} aesthetic. Use this skill when reproducing, extending, or building new screens that match the {product-name} design system. Works with any tech stack — specify your stack when invoking."
---

# {Product Name} UI Design System

This skill encodes the complete design system of {product-name}, extracted from {N} UI screenshots. It provides everything needed to faithfully reproduce or extend the product's visual language in any technology stack.

## How to Use This Skill

1. Reference this skill when building new screens or components for {product-name}
2. Specify your target tech stack (React, Next.js, Vue, Tailwind, SwiftUI, Flutter, etc.)
3. For component generation, this skill pairs with `ui-ux-pro-max` for production-quality output
4. All design decisions should trace back to the tokens in `references/design-tokens.md`

## Design System Summary

- **Pages analyzed**: {list page names}
- **Visual style**: {primary style(s)}
- **Color palette**: {N} colors — primary: {primary color hex}
- **Typography**: {font family}, {N} sizes, weights {weight range}
- **Components**: {N} components identified
- **Spacing base**: {base unit}px grid

## Reference Files

- `references/design-tokens.md` — Colors, typography, spacing, radius, shadows
- `references/components.md` — Component inventory with variants and states
- `references/layout.md` — Page layouts and navigation structure
- `references/style-guide.md` — Visual style, interaction patterns, UX principles

## Quick Start

To build a new screen matching this design system:

```
Using the {product-name}-ui skill:
- Apply the color tokens from design-tokens.md
- Use the component specs from components.md
- Follow the layout patterns from layout.md
- Match the visual style defined in style-guide.md

Target stack: [specify your stack]
Page to build: [describe the page]
```

## Generating Code

When generating code from this skill:
1. Load `references/design-tokens.md` — these are the source of truth for all visual values
2. Identify which components from `references/components.md` are needed
3. Apply the layout pattern from `references/layout.md` that matches the target page type
4. Invoke `ui-ux-pro-max` for production-quality component generation with these tokens as context
```
